# 테스트 및 반복 개선

Skill은 필요에 따라 다양한 수준의 엄격함으로 테스트할 수 있습니다:

- **Claude.ai에서 수동 테스트** — 쿼리를 직접 실행하고 동작을 관찰합니다. 빠른 반복이 가능하고 별도 설정이 필요 없습니다.
- **Claude Code에서 스크립트 테스트** — 변경 사항에 대해 반복 가능한 검증을 위해 테스트 케이스를 자동화합니다.
- **Skills API를 통한 프로그래매틱 테스트** — 정의된 테스트 세트에 대해 체계적으로 실행되는 평가 스위트를 구축합니다.

품질 요구사항과 Skill의 가시성에 맞는 접근 방식을 선택하세요.

> **Pro Tip: 확장하기 전에 단일 작업에 대해 반복 개선하세요.** 가장 효과적인 Skill 제작자는 Claude가 성공할 때까지 하나의 어려운 작업에 대해 반복 개선한 다음, 성공한 접근 방식을 Skill로 추출합니다. 이는 Claude의 in-context learning을 활용하며 광범위한 테스트보다 더 빠른 신호를 제공합니다. 작동하는 기반이 마련되면, 커버리지를 위해 여러 테스트 케이스로 확장합니다.

## 권장 테스트 접근법

### 1. 트리거 테스트

**목표:** Skill이 적절한 시점에 로드되는지 확인합니다.

테스트 케이스:
- ✅ 명확한 작업에서 트리거됨
- ✅ 바꿔 말한 요청에서도 트리거됨
- ❌ 관련 없는 주제에서는 트리거되지 않음

테스트 스위트 예시:

```text
트리거되어야 하는 경우:
- "Help me set up a new ProjectHub workspace"
- "I need to create a project in ProjectHub"
- "Initialize a ProjectHub project for Q4 planning"

트리거되지 않아야 하는 경우:
- "What's the weather in San Francisco?"
- "Help me write Python code"
- "Create a spreadsheet" (ProjectHub skill이 시트를 처리하지 않는 한)
```

### 2. 기능 테스트

**목표:** Skill이 올바른 결과를 생성하는지 검증합니다.

테스트 케이스:
- 유효한 결과물 생성
- API 호출 성공
- 오류 처리 작동
- 엣지 케이스 커버

예시:

```text
Test: 5개의 태스크가 포함된 프로젝트 생성
Given: 프로젝트 이름 "Q4 Planning", 5개의 태스크 설명
When: Skill이 워크플로우를 실행
Then:
  - ProjectHub에 프로젝트 생성됨
  - 올바른 속성으로 5개 태스크 생성됨
  - 모든 태스크가 프로젝트에 연결됨
  - API 오류 없음
```

### 3. 성능 비교

**목표:** Skill이 기준선 대비 결과를 개선하는지 증명합니다.

**Skill 없이:**
- 사용자가 매번 지침을 제공
- 15번의 대화 왕복
- 재시도가 필요한 API 호출 실패 3건
- 12,000 토큰 소비

**Skill과 함께:**
- 자동 워크플로우 실행
- 확인 질문 2개만
- API 호출 실패 0건
- 6,000 토큰 소비

## skill-creator Skill 활용

skill-creator skill은 Claude.ai의 플러그인 디렉토리에서 이용하거나 Claude Code용으로 다운로드할 수 있으며, Skill을 구축하고 반복 개선하는 데 도움을 줍니다.

**Skill 생성:**
- 자연어 설명으로부터 Skill 생성
- 적절한 형식의 frontmatter가 포함된 SKILL.md 생성
- 트리거 문구와 구조 제안

**Skill 리뷰:**
- 일반적인 문제 지적 (모호한 description, 누락된 트리거, 구조적 문제)
- 과다/과소 트리거 위험 식별
- Skill의 명시된 목적에 기반한 테스트 케이스 제안

**반복 개선:**
- Skill을 사용하면서 엣지 케이스나 실패를 만나면, 해당 예시를 skill-creator에 다시 가져옵니다
- 예시: "이 채팅에서 확인된 이슈와 해결책을 사용하여 [특정 엣지 케이스]를 처리하는 방식을 개선해 주세요"

**사용 방법:**
```text
"Use the skill-creator skill to help me build a skill for [your use case]"
```

참고: skill-creator는 Skill 설계와 개선을 도와주지만, 자동화된 테스트 스위트를 실행하거나 정량적 평가 결과를 생성하지는 않습니다.

## 피드백 기반 반복 개선

Skill은 살아있는 문서입니다. 다음을 기반으로 반복 개선을 계획하세요:

### 과소 트리거 신호
- Skill이 활성화되어야 할 때 로드되지 않음
- 사용자가 수동으로 활성화함
- 사용 시점에 대한 지원 문의 발생

**해결책:** description에 더 많은 세부사항과 뉘앙스를 추가하세요 — 특히 기술 용어에 대한 키워드를 포함할 수 있습니다.

### 과다 트리거 신호
- 관련 없는 쿼리에 Skill이 로드됨
- 사용자가 비활성화함
- 목적에 대한 혼란 발생

**해결책:** 부정 트리거를 추가하고, 더 구체적으로 작성하세요.

### 실행 문제
- 일관되지 않은 결과
- API 호출 실패
- 사용자 수정 필요

**해결책:** 지침을 개선하고, 오류 처리를 추가하세요.
