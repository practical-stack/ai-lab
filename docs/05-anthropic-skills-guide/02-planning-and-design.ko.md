# 기획 및 설계

## Use Case부터 시작하기

코드를 작성하기 전에, Skill이 지원해야 할 2~3가지 구체적인 Use Case를 정의하세요.

**좋은 Use Case 정의:**

```text
Use Case: Project Sprint Planning
Trigger: 사용자가 "이번 스프린트 기획 도와줘" 또는 "스프린트 태스크 생성"이라고 말할 때
Steps:
  1. Linear에서 현재 프로젝트 상태를 가져옴 (MCP 활용)
  2. 팀 속도와 가용 인력 분석
  3. 태스크 우선순위 제안
  4. 적절한 라벨과 추정치를 포함하여 Linear에 태스크 생성
Result: 태스크가 생성된 완전한 스프린트 계획
```

스스로에게 물어보세요:
- 사용자가 달성하고자 하는 것은 무엇인가?
- 이를 위해 어떤 다단계 워크플로우가 필요한가?
- 어떤 도구가 필요한가 (내장 또는 MCP)?
- 어떤 도메인 지식이나 모범 사례가 내장되어야 하는가?

## 일반적인 Skill Use Case 카테고리

### 카테고리 1: 문서 및 에셋 생성

**용도:** 문서, 프레젠테이션, 앱, 디자인, 코드 등 일관되고 고품질의 결과물 생성.

**실제 예시:** frontend-design skill (docx, pptx, xlsx, ppt용 Skill도 참고)

> "높은 디자인 품질로 독창적이고 프로덕션 수준의 프론트엔드 인터페이스를 생성합니다. 웹 컴포넌트, 페이지, 아티팩트, 포스터 또는 애플리케이션을 만들 때 사용하세요."

**핵심 기법:**
- 스타일 가이드 및 브랜드 표준 내장
- 일관된 결과물을 위한 템플릿 구조
- 최종 확정 전 품질 체크리스트
- 외부 도구 불필요 — Claude의 내장 기능 활용

### 카테고리 2: 워크플로우 자동화

**용도:** 여러 MCP 서버 간 조율을 포함하여, 일관된 방법론이 필요한 다단계 프로세스.

**실제 예시:** skill-creator skill

> "새로운 Skill을 생성하기 위한 대화형 가이드. Use Case 정의, frontmatter 생성, 지침 작성, 검증 과정을 사용자와 함께 진행합니다."

**핵심 기법:**
- 검증 단계가 포함된 단계별 워크플로우
- 일반적인 구조를 위한 템플릿
- 내장된 리뷰 및 개선 제안
- 반복 개선 루프

### 카테고리 3: MCP 강화

**용도:** MCP 서버가 제공하는 도구 접근에 워크플로우 가이드를 추가.

**실제 예시:** sentry-code-review skill (Sentry 제공)

> "Sentry의 MCP 서버를 통해 에러 모니터링 데이터를 활용하여 GitHub Pull Request에서 감지된 버그를 자동으로 분석하고 수정합니다."

**핵심 기법:**
- 여러 MCP 호출을 순차적으로 조율
- 도메인 전문지식 내장
- 사용자가 별도로 지정해야 할 컨텍스트 제공
- 일반적인 MCP 이슈에 대한 오류 처리

## 성공 기준 정의

Skill이 잘 작동하는지 어떻게 알 수 있을까요? 이것들은 정밀한 임계값이 아닌, 대략적인 벤치마크로서의 목표 수치입니다.

### 정량적 지표

- **관련 쿼리의 90%에서 Skill이 트리거됨**
  - 측정 방법: Skill을 트리거해야 하는 테스트 쿼리 10~20개를 실행합니다. 자동으로 로드되는 횟수 대비 명시적으로 호출해야 하는 횟수를 추적합니다.
- **X번의 tool call로 워크플로우 완료**
  - 측정 방법: Skill 활성화 여부에 따라 동일한 작업을 비교합니다. Tool call 횟수와 총 토큰 소비량을 셉니다.
- **워크플로우당 API 호출 실패 0건**
  - 측정 방법: 테스트 실행 중 MCP 서버 로그를 모니터링합니다. 재시도율과 오류 코드를 추적합니다.

### 정성적 지표

- **사용자가 Claude에게 다음 단계를 안내할 필요가 없음**
  - 평가 방법: 테스트 중 리다이렉트하거나 명확히 해야 하는 빈도를 기록합니다. 베타 사용자에게 피드백을 요청합니다.
- **사용자 수정 없이 워크플로우 완료**
  - 평가 방법: 동일한 요청을 3~5회 실행합니다. 구조적 일관성과 품질 면에서 결과물을 비교합니다.
- **세션 간 일관된 결과**
  - 평가 방법: 새로운 사용자가 최소한의 안내로 첫 시도에 작업을 완수할 수 있는가?
