# 기본 개념

## Skill이란?

Skill은 Claude에게 특정 작업이나 워크플로우를 처리하는 방법을 알려주는 지침 세트로, 간단한 폴더 형태로 패키징됩니다. 매 대화마다 선호도, 프로세스, 도메인 전문지식을 다시 설명하는 대신, Skill을 통해 한 번만 가르치면 매번 그 혜택을 누릴 수 있습니다.

Skill은 다음으로 구성된 폴더입니다:

- **SKILL.md** (필수): YAML frontmatter가 포함된 Markdown 형식의 지침
- **scripts/** (선택): 실행 가능한 코드 (Python, Bash 등)
- **references/** (선택): 필요 시 로드되는 문서
- **assets/** (선택): 출력에 사용되는 템플릿, 폰트, 아이콘

## 핵심 설계 원칙

### Progressive Disclosure

Skill은 세 단계 시스템을 사용합니다:

1. **첫 번째 단계 (YAML frontmatter):** Claude의 system prompt에 항상 로드됩니다. 전체 내용을 컨텍스트에 로드하지 않고도 각 Skill을 언제 사용해야 하는지 Claude가 판단할 수 있을 정도의 정보만 제공합니다.
2. **두 번째 단계 (SKILL.md 본문):** Claude가 현재 작업과 관련이 있다고 판단할 때 로드됩니다. 전체 지침과 가이드가 포함되어 있습니다.
3. **세 번째 단계 (링크된 파일):** Skill 디렉토리 내에 번들된 추가 파일로, Claude가 필요할 때만 탐색하여 참조합니다.

이러한 Progressive Disclosure를 통해 전문적인 역량을 유지하면서도 토큰 사용을 최소화할 수 있습니다.

### 조합성 (Composability)

Claude는 여러 Skill을 동시에 로드할 수 있습니다. 여러분의 Skill은 자신이 유일한 기능이라고 가정하지 않고, 다른 Skill과 잘 어울려 동작해야 합니다.

### 이식성 (Portability)

Skill은 Claude.ai, Claude Code, API 어디서든 동일하게 작동합니다. 한 번 만들면 환경이 해당 Skill의 의존성을 지원하는 한 수정 없이 모든 플랫폼에서 사용할 수 있습니다.

## MCP 개발자를 위한 안내: Skills + Connectors

이미 작동하는 MCP 서버가 있다면, 가장 어려운 부분은 끝났습니다. Skill은 그 위에 올라가는 지식 계층으로, 여러분이 이미 알고 있는 워크플로우와 모범 사례를 캡처하여 Claude가 일관되게 적용할 수 있게 합니다.

### 주방 비유

- **MCP**는 전문 주방을 제공합니다: 도구, 재료, 장비에 대한 접근 권한.
- **Skills**는 레시피를 제공합니다: 가치 있는 결과물을 만들어내기 위한 단계별 지침.

함께 사용하면, 사용자가 모든 단계를 직접 파악할 필요 없이 복잡한 작업을 수행할 수 있게 됩니다.

### 함께 작동하는 방식

| MCP (연결성) | Skills (지식) |
|---|---|
| Claude를 서비스(Notion, Asana, Linear 등)에 연결 | Claude에게 서비스를 효과적으로 사용하는 방법을 교육 |
| 실시간 데이터 접근 및 도구 호출 제공 | 워크플로우와 모범 사례 캡처 |
| Claude가 **할 수 있는** 것 | Claude가 **해야 하는** 방법 |

### MCP 사용자에게 이것이 중요한 이유

**Skill 없이:**
- 사용자가 MCP를 연결하지만 다음에 무엇을 해야 할지 모름
- "연동 기능으로 X를 어떻게 하나요"라는 지원 티켓 발생
- 매 대화가 처음부터 시작됨
- 사용자마다 프롬프트가 달라 결과가 일관되지 않음
- 실제 문제는 워크플로우 가이드 부족인데 커넥터를 탓하게 됨

**Skill과 함께:**
- 사전 구축된 워크플로우가 필요할 때 자동으로 활성화
- 일관되고 신뢰할 수 있는 도구 사용
- 모든 상호작용에 모범 사례가 내장
- 연동 기능의 학습 곡선 감소
